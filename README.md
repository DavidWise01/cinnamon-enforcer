> *How AI Flattens Everything You Say — and Why the Sharp Edges Matter.*

Every language model carries a **synonym chip**. You type *"furious"*; it returns *"very upset."* You write *"the denial letter was self-contradictory"*; it gives back *"there may have been an inconsistency."* It doesn't censor or refuse — it **rounds**. Specific → general, strong → weak, evidence → vagueness, your words → the model's words, politely, with excellent grammar. And cinnamon becomes "warm spice."

The Cinnamon Enforcer is the stance that puts the sharp thing back.

## The flattening (six types)

| type | example |
|---|---|
| Vocabulary | `furious` → angry |
| Register | `denial pursuant to exclusion` → they said no |
| Evidence | `the March 19 letter from Kornovich` → their response |
| Authority | `§72A.201 Subd.8(1)` → state law |
| Certainty | `it contradicts itself` → there may be an issue |
| Agency | `Swan closed the file` → the file was closed |

In casual talk it's survivable. In **law, medicine, engineering, and evidence**, a synonym changes a duty, a diagnosis, a spec, a verdict.

## The protocol

1. **Name the specific thing** (a model can't synonym-substitute a proper noun).
2. **Reject the hedge** — "use my words."
3. **Anchor to evidence** (document, date, person, citation — the un-flattenable).
4. **Preserve your register.**
5. **Audit the output** — did my words, citations, certainty, attributions survive?
